Elemento Headings <h1> - <h6>

Tenemos 6 niveles de Headings, los utilizamos para indicar la

prioridad de los encabezados, van de mayor interés a menor interés.

Este es un encabezado de primer nivel

Este es un encabezado de segundo nivel

        
          <h2>Este es un encabezado de segundo nivel</h2>

Este es un encabezado de tercer nivel

        
          <h3>Este es un encabezado de tercer nivel</h3>

Este es un encabezado de cuarto nivel

        
          <h4>Este es un encabezado de cuarto nivel</h4>
Este es un encabezado de quinto nivel
        
          <h5>Este es un encabezado de quinto nivel</h5>
Este es un encabezado de sexto nivel
        
          <h6>Este es un encabezado de sexto nivel</h6>

Elemento Párrafo <p>

Se utiliza cuando queremos ingresar texto

Este es un párrafo

        
          <p>Este es un párrafo</p>

Dentro de los párrafos podemos usar elementos para dar importancia, relevancia o énfasis a una palabra en especifico

Esto es un EJEMPLO de resaltado

Esto es un EJEMPLO de importancia

        
          <p>Esto es un <b>EJEMPLO</b> de resaltado</p>
          <p>Esto es un <strong>EJEMPLO</strong> de importancia</p>

Dentro de los párrafos también podemos usar enlaces para que, si hacemos clic, en una palabra

en específico nos envié a una página o una referencia, externa a la página o dentro de la misma pagina.

Ejemplo:

Si desea saber más sobre los elementos que se pueden utilizar en los párrafos HAGA CLICK AQUI

        
          <p>Si desea saber más sobre los elementos que se pueden utilizar en los párrafos <a href="https://developer.mozilla.org/en-US/docs/Web/HTML">HAGA CLICK AQUI</a></p>

Elemento Listas

Se utilizan cuando queremos enlistar algo, y tenemos 3 tipos:

  • Ordenadas
  • Desordenadas
  • De definiciones

Listas Ordenadas

  1. Elemento#1
  2. Elemento#2
  3. Elemento#3
  4. Elemento#4
        
          <ol>
            <li>Elemento#1</li>
            <li>Elemento#2</li>
            <li>Elemento#3</li>
            <li>Elemento#4</li>
          </ol>

Listas Desordenadas o sin numeración

  • Elemento#1
  • Elemento#2
  • Elemento#3
  • Elemento#4
        
          <ul>
            <li>Elemento#1</li>
            <li>Elemento#2</li>
            <li>Elemento#3</li>
            <li>Elemento#4</li>
          </ul>

Hipertext Markup Language

Cascading Style Sheets

Hypertext Transfer Protocol

o periódico, o una entrada de blog, etc.

Etiqueta <section>

Utilizamos esta etiqueta para representar una sección independiente genérica de un documento, que no tiene un elemento

semántico más específico para representarla. Las secciones siempre deben tener un encabezado, con muy pocas excepciones.

Ejemplo:

Elegir una manzana

Introducción

Este documento proporciona una guía para ayudar con la importante tarea de elegir una manzana correcto.

Criterios

Hay muchos criterios diferentes a tener en cuenta al elegir una manzana: tamaño, color, firmeza, dulzura, acidez...

      
        <h3>Elegir una manzana</h3>
        <section>
          <h3>Introducción</h3>
          <p>Este documento proporciona una guía para ayudar con la importante tarea de elegir una manzana correcto.</p>
        </section>
    
        <section>
          <h3>Criterios</h3>
          <p>Hay muchos criterios diferentes a tener en cuenta al elegir una manzana: tamaño, color, firmeza, dulzura, acidez...</p>
       </section>

Elementos Formularios

Tenemos varios elementos para realizar formularios.
Algunos de estos Son:

  • input y label ("text", "checkbox" y "radio")
  • fieldset y legend
  • select
  • textarea

Input y label

Input:Este elemento se usa para crear controles interactivos para formularios para aceptar datos del usuario.
label:Este elemento representa un título para un elemento en una interfaz de usuario.

Tipo "Text"

      
        <form>
          <label for="nom">Nombre
            <input type="text" name="nom" id="nom">
          </label>
        </form>

Tipo "checkbox"

      
        <form>
          <label for="remo">Remo
            <input type="checkbox" name="encues" id="remo">
          </label>
          <label for="judo">Judo
            <input type="checkbox" name="encues" id="judo">
          </label>
        </form>

Tipo "Radio"

      
        <form>
          <label for="azul">Azul
            <input type="radio" name="colorfavorito" id="azul">
          </label>
          <label for="verde">Verde
            <input type="radio" name="colorfavorito" id="verde">
          </label>
        </form>

Fieldset y Legend

Fieldset:Este elemento se utiliza para agrupar varios controles, así como etiquetas (<label>) dentro de un formulario web.
Legend:Este elemento representa un título para el contenido de su padre <fieldset>.

Color Favorito}

            </label>

            </label>
            <br>
            <label for="rosado">Rosado
              <input type="radio" name="colorfavorito" id="rosado">
            </label>
          </fieldset>
        </form>

Select

Este elemento representa un control que proporciona un menú de opciones. 

      
        <form>
          <label for="nombre-pet">Elija un nombre:</label>
          <select name="nombre-pets" id="nombre-pet">
            <option value="#">Por favor elija un nombre para su mascota</option>
            <option value="#">Noah</option>
            <option value="#">Pebels</option>
            <option value="#">Dante</option>
            <option value="#">Taquito</option>
            <option value="#">Spider</option>
            <option value="#">Rico</option>
          </select>
        </form>

Textarea

Este elemento representa un control de edición de texto sin formato de varias líneas,
útil cuando desea permitir que los usuarios ingresen una cantidad considerable de texto de forma
libre, por ejemplo, un comentario en una revisión o formulario de comentarios. 

      
        <form>
          <label for="coment">Dejanos tus sugerencias:</label>
          <textarea id="coment" name="dialogue" rows="7" cols="35">
            Pueden mejorar el servicio al cliente por...
          </textarea>
        </form>
